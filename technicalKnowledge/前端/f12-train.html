玩转F12---chrome开发者工具

开发者工具十分强大，是从事前端开发工作的必备神器，可以这么说，95%以上的前端问题都可以通过工具找到问题并解决，不推荐通过源码去查找问题。不会使用工具，就像电工不会看电表一样，让你无所适从。

一、Elements

  • Element主面板

    展示整个document文档树，在这里可以看到整个文档的结构。

    🌈小提示：用户可通过点击某个dom确定当前控制台变量$0所对应的dom对象。

  • Styles

    样式面板由三部分组成：

    • 自己定义的样式: xxx.css

    • 继承自父类的样式: inherited form XXX

    • 浏览器自带的默认样式: user agent stylesheet

    :hov 可以让你轻松调试基于伪类编写的样式

    .cls 可以管理所有样式class

  • computed

    通过样式计算，权重分析，得到确定样式，该样式将是最终应用到浏览器上，所以conputed显示的值都是确定值。

    • 看懂盒子模型

    • 看懂样式的生效方式

      主动 被动的区分

      权重优先级

    • Show all 查看应用到dom的所有样式

    🌈扩展结论：

    每个dom都具备所有样式，只是有的被用户定义，有的来自继承，有的是系统默认，最终样式由CSSOM计算所得，并与dom合并为渲染树render Tree，进行布局Layout，绘制Paint，并最终呈现效果 [^tag1]。

  • Layout

    展示页面哪些应用的flex弹性布局和grid网格布局

  • Event Listeners

    查看绑定在该dom下的所有事件监听。

    通过查看面板，

    也可以通过Api getEventListeners(dom)获得。

  • DOM Breakpoints

    用于监听dom变化，任何dom改动都将触发断点，从而定位引起dom变化发生的位置。

    dom断点有3种类型:

    自身属性发生变化: attribute modifications

    子孙后代发生变化: subtree modifications

    自身dom被删除： node removal

    有时一些框架如Vue、React，通过双向绑定操作dom的，用dom断点可以轻松定位问题的触发点，通过调用栈轻松锁定问题发生的原因。

  • DOM Properties

    查看Dom对象的所有属性。

二、Console 控制台

  • Console Api

    console提供了十几种api，下面列出一些平时比较经常用到的，如下：

    1. 消息类

      //按消息级别
console.log("Here is log");
console.info("Here is info");
console.debug("Here is debug");
console.warn("Here is warn");
console.error("Here is error");
//可以同时打印多个
console.log(1,2,3);
//占位符
var firstName = "Maple", familyName = "Jan";
console.log("My name is %s %s", firstName, familyName); 
//css 样式
console.log('%cThis is a css text', "color:white;background-color: blue");

    2. 打印复杂对象

      //以表格形式打印对象
console.table(location)
//以对象结构打印
console.dir(document.body)

    3. 计数与计时

      //计数
for(var i=0;i<999;i++) {
    console.count("aaa");
}
//结束
console.countReset("aaa")

//计时
console.time("times");
var n = 0;
for(var i = 0; i < 9999999; i++) {
    n += 2;
};
console.timeEnd("times")

    4. 分组

      //分组
console.group("1");
console.log("1.1");
console.log("1.2");
console.log("1.3");
//分组+折叠
console.groupCollapsed("1.4");
console.log("1.4.1");
console.log("1.4.2");
//结束分组
console.groupEnd();

    5. 跟踪堆栈

      function b(){
    console.log("b");
    //有时候在排查到底是哪里调用了它时，非常有用
    console.trace();
}

function a(){
    console.log("a")
    b(); 
}
a();

    6. 其他

      //断言--测试
function sum(a,b){ return a+b }
console.assert(sum(1,2) === 3);
//清除控制台
console.clear();

  • console 控制台内置工具

    1. $0 - $4

      代表选择的dom，最新为$0，次新为$1，以此类推，一直到$4，为最近选择的5个dom元素。

      //判断是否包含子元素
$1.contains($0)
//简化jquery selector写法
$($0).val(123);
//直接操作dom元素
$0.style.color = "red";

    2. $_

      $_表示上一次控制台运行程序的返回值

      > $("input")
> $_.length

    3. $()

      $() 等价于document.querySelector()

      注意：jquery、mootools应用下$变量可能被覆盖

    4. $$()

      $$() 等价于document.querySelectorAll()

⚠️注意：以上4种用法，只允许在控制台下用于简化调试的快捷方式，实际在项目的js环境里是不存在的，因此不应该在项目代码中使用。

  • console toolbar工具说明

    1. 切换window作用域（上下文）

    2. 实时的表达式计算（用于监控某个dom、js变量实时变化或某个程序运行结果）

    3. 阻止控制台清空历史

    🌈小提示：无处不在的console，在非console面板下，可以通过按键 esc 切换控制台的显示或隐藏，以配合其他面板使用。

三、Sources 源码

  • 资源管理器

    显示top下，所有站点的所有资源，包括网页、静态资源，第三方cdn、iframe等。

  • 调试技巧

    1. 搜索源代码 ctrl + shift + f，锁定位置，并进行断点调试。

      快速打开源文件 ctrl + p

    2. 条件断点 设置条件，在条件满足的时候自动打上断点

      function showName(_, index){
   console.log("showName", index);
}

Array(100).fill(1).forEach(showName)

    3. 异常断点

      自动在异常处打上断点，可以快速定位到异常发生前的上下文

      //填充’a‘到一百个数组成员 
var array = Array(100).fill('a'); 
array[50] = 50; 
array.forEach(function(item, index){ 
     var item = item + index; 
    //将在发生异常的第50个成员处自动设上断点             
    console.log(item.replace('a', 'A')) 
});

    4. 添加日志点 + 禁用断点（部分）

      添加log打印位置：add logpoint，输入要打印的变量，无需通过修改源代码，即可实现动态打印的console.log的功能；

      禁用断点：禁用所有断点，禁用部分断点。

    5. 线上调试源码

      Sources Overrides 用于线上调试源码，支持本地修改，并在线上生效。

      不仅支持js/css，对网页源码也能支持本地修改。

      原理：开启overrides后，通过编辑本地资源，浏览器会优先使用本地资源，并进行线上调试，在排除问题后，把改动生效到生产环境即可。

    6. 灵活使用watch

      通过添加表达式，实时监控某些变量或表达式的计算结果，对于断点调试起到很好的辅助效果。

    7. 看懂Scope / Call Stack

      Scope 程序运行时的作用域，能使用的变量，包括局部和全局。

      Call Stack 看懂调用栈，对调试程序事半功倍。

    8. XHR/fetch Breakpoints

      设置监控的url，程序会在即将发起请求时，自动打上断点，从而方便调试。

    9. Run command

      通过 ctrl + shift + p 可以快速运行run command

      运行command用法有很多，具体看使用场景，如：

      禁用javascript

      禁用sourceMap

      打印工具/监视器

  • 关于脚本生产版本的说明

    正常情况下，对于线上服务都会引入经过打包压缩，通过减少体积，提高传输性能，通过压缩，提高解析性能。

    1. 传统第三方库，一般区分开发版本和生产版本，通常以 xxx.js/xxx.min.js加以区分。

      为了方便调试，需要在开发阶段引入开发版本，而在生产阶段引入生产版本，但传统的web开发模式不支持env环境变量，也不涉及打包编译，因此使用上需要通过后端做区分环境，并根据不同的env引入不同版本的js。

    2. sourceMap 通过源码地图，链接到指定的map文件，该文件记录了所有源码坐标，并在打印压缩脚本时，自动引入map，并进行源代码的展示。

      一般生产版本与map文件同目录，并且在生产版本js的末尾有如下代码：

      //# sourceMappingURL=app.js.map

      用于指定该js对应的map，当进行断点调试时，map文件会映射到源码版本，以方便用户调试程序。

    3. Pretty formatter

      在不具备sourceMap的前提下，通过pretty可以快速格式化生产版本，以达到快速锁定目标行，并进行调试程序的目的。
